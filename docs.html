1. Your service must be packaged as a Docker image.
A Dockerfile/Containerfile:

  • - Must expose a port with EXPOSE (for now only TCP is supported).
  • - Should have a HEALTHCHECK to ensure users are only redirect once the service is ready.
Dockerfile/Containerfile example, assuming an index.html file at the root directory:
FROM docker.io/busybox COPY . /var/www EXPOSE 8080 HEALTHCHECK CMD nc -zw 1 ::1 8080 # Workaround only for this image to prevent hang-up STOPSIGNAL 9 CMD /bin/httpd -vvfp 8080 -h /var/www

2. Store the API secrets. See here for GitHub Actions.

3. Build then upload the image to the Caasis API.
It is recommended to upload a unique image name, by either using the image ID (which are like `a1b2c3d4`) or a unique image tag to avoid potential collusions on concurrent uploads.


Local upload

docker build . -o - | gzip | curl -Lu "$CAASIS_API_KEY" --data-binary @- http://api.caasis.cloud/new

GitHub Actions example

name: Build on: push: branches: [master, main] pull_request: types: [opened, reopened, synchronize] permissions: pull-requests: write jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v6 - name: Set up Docker Buildx uses: docker/setup-buildx-action@v3 - name: Build run: docker buildx build . -o type=tar,dest=image.tar - name: Push image to Caasis run: |- gzip -c image.tar | curl -Lu "$CAASIS_API_KEY" --data-binary @- https://api.caasis.cloud/new | gh pr comment --create-if-none --edit-last ${{ github.head_ref }} -F - if: ${{ github.head_ref }} env: GH_TOKEN: ${{ github.token }} CAASIS_API_KEY: ${ secrets.CAASIS_API_KEY }

See also GitHub documentation about gh pr comment

4. Go to the returned URL link, your service will be standing-by and start on the first request, up for 15 minutes.
